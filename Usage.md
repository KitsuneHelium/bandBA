## 1. 快速开始

### 1.1 运行环境
- 操作系统：Windows 10/11（推荐）
- Python版本：Python 3.8 或更高版本
- 唯一依赖：PyQt5

### 1.2 安装与启动

#### 方法一：直接运行（推荐）
1.  下载仓库中的 `vge_editor.py` 文件
2.  打开命令提示符，安装依赖：
pip install PyQt5
plaintext
3.  双击运行 `vge_editor.py` 即可启动编辑器

#### 方法二：打包为EXE（无需Python环境）
如果你想分享给没有Python环境的朋友，可以使用PyInstaller打包：
pip install pyinstallerpyinstaller -F -w vge_editor.py
plaintext
打包完成后，在 `dist` 文件夹中可以找到独立的EXE文件。

## 2. 界面介绍

编辑器采用三栏式布局，直观高效：

### 2.1 左侧：流程图分支编辑器
- 蓝色节点：代表 Label（剧情分支入口）
- 彩色节点：代表具体的游戏指令（对话、背景、立绘等）
- 拖拽节点会自动吸附对齐（60px网格）
- 支持右键菜单快速添加节点

### 2.2 中间：完整剧本JSON代码区
- 实时同步显示流程图对应的JSON源代码
- 支持直接编辑代码，修改会自动同步到流程图
- 按 F5 可以强制从代码重新生成流程图
- 代码自动格式化，缩进清晰

### 2.3 右侧：实时渲染预览与快捷指令
- 游戏实时渲染预览：1:1模拟小米手环9 Pro的显示效果
- 快捷指令栏：一键添加所有常用剧本指令
- 支持在预览器中点击选项进行交互测试

## 3. 基础操作

### 3.1 首次使用必做：设置工作目录
这是最重要的一步，否则图片选择功能将无法正常工作：

1.  点击顶部菜单栏 文件 → 设置工作目录
2.  选择你本地 bandBA 项目中的 `src/common` 文件夹
3.  点击确定，重启编辑器生效

> 提示：工作目录设置后，编辑器会自动识别所有背景和立绘图片，无需手动输入路径。

### 3.2 创建你的第一个剧本

#### 步骤1：添加Label（剧情分支）
1.  在左侧流程图空白处右键点击
2.  选择 添加Label节点
3.  双击蓝色的Label节点可以重命名（如"第一章"、"真寻线"等）

#### 步骤2：添加游戏指令
有三种便捷的添加方式：
- 右键菜单：在任意位置添加新节点
- 快捷按钮：快速添加常用指令
- 选中追加：在选中节点下方插入新指令

推荐操作流程：
1.  先选中一个已有的节点
2.  点击右侧快捷按钮或使用右键菜单
3.  新指令会自动插入到选中节点的下方

#### 步骤3：编辑指令参数
1.  双击任意一个彩色指令节点
2.  在弹出的对话框中修改参数
3.  点击确定保存，节点标题会自动更新

### 3.3 图片资源管理

#### 可视化选择背景/立绘
1.  先添加一个 bg 或 chara 节点
2.  右键点击该节点
3.  选择 选择背景图片 或 选择立绘图片
4.  在弹出的窗口中：
    - 顶部下拉框选择子文件夹
    - 中间区域双击图片即可选中
5.  编辑器会自动将本地路径转换为游戏内的 `/common/` 路径

### 3.4 保存与导出

#### 保存工程文件
- 点击 文件 → 保存 或按 Ctrl+S
- 会保存为 .json 格式的工程文件，方便以后继续编辑

#### 导出游戏剧本
1.  点击 文件 → 导出游戏剧本 或右侧的 导出游戏剧本 按钮
2.  将文件命名为 `gamedata1.txt`
3.  直接放入游戏的 `/Common/GAMEDATA/` 目录即可运行

## 4. 进阶技巧

### 4.1 实时预览功能

#### 选中即预览
- 点击任意指令节点，右侧预览器会立即显示该节点在游戏中的效果
- 支持预览背景、立绘、对话、选项等所有内容

#### 模拟游戏流程
顶部工具栏提供了三个预览控制按钮：
- 上一步：回退到上一条指令
- 重置预览：清空预览器
- 下一步：模拟点击屏幕，继续剧情

### 4.2 撤销与重做
- 按 Ctrl+Z 撤销操作
- 按 Ctrl+Y 重做操作
- 编辑器会自动保存最多 50 步历史记录，放心大胆地编辑

### 4.3 连接Label（自动生成跳转）
1.  按住 Ctrl 键，同时选中两个蓝色的 Label 节点
2.  右键点击
3.  选择 连接Label（生成goto）
4.  会自动在第一个 Label 的末尾添加 goto 指令跳转到第二个 Label

### 4.4 选项分支编辑
1.  添加一个 select 节点
2.  双击编辑
3.  按照以下格式输入：
选项 1 -> label1选项 2 -> label2
plaintext
每行一个选项，`->` 前面是选项文本，后面是跳转的目标 Label 名称。

### 4.5 代码自动定位
- 当你在流程图中选中一个节点时，中间的JSON代码区会自动滚动到对应的代码行
- 方便你进行精细的代码调整

## 5. 指令参考

### 5.1 完整指令列表

| 指令类型 | 节点颜色 | 作用 |
|---------|---------|------|
| Label | 蓝色 | 剧情分支入口 |
| bg | 橙色 | 切换背景图片 |
| chara | 绿色 | 切换人物立绘 |
| text | 红色 | 显示对话文本 |
| ani | 黄色 | 播放人物动画 |
| bg_ani | 黄色 | 播放背景动画 |
| wait | 灰色 | 等待用户点击 |
| select | 亮蓝色 | 显示选项分支 |
| goto | 紫色 | 跳转到另一个 Label |
| warp | 深紫色 | 跳转到另一个剧本文件 |
| end | 深红色 | 剧情结束 |

### 5.2 动画参数说明

在 ani 或 bg_ani 指令中，可以使用以下动画名称：
- jump：上下跳动
- shakex：左右抖动
- shakey：上下抖动

## 6. 小贴士

1.  自动保存：编辑器每30秒会自动保存一次当前文件，防止意外崩溃丢失进度
2.  深色模式：默认采用护眼深色主题，长时间编写剧本不累眼
3.  快捷键：
- Ctrl+N：新建文件
- Ctrl+O：打开文件
- Ctrl+S：保存文件
- Ctrl+Z：撤销
- Ctrl+Y：重做
- F5：刷新流程图

---

## 7. 注意事项

- 请确保工作目录设置正确，否则图片路径可能无法正常转换
- 导出的剧本文件请命名为 `gamedata1.txt` 并放入正确的目录
- 本编辑器生成的剧本完全兼容 bandBA 引擎
